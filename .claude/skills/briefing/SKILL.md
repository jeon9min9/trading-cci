# 아침 브리핑

미국장 마감 후 관심 종목(SOXL, BITX, TQQQ, SPXL, QLD)의 상태를 정리한다.

## 절차

1. `data/positions.json` 읽기 (프로젝트 루트 기준). 없으면 "포지션 데이터 없음 — 웹앱 설정 화면에서 내보내기 후 붙여넣어달라"고 안내하고 관심 종목 시세만 진행.
2. 종목별 전일 종가·등락률·CCI(20, 일봉) 수집:
   - Alpha Vantage MCP가 연결되어 있으면 그 도구 사용 (무료 25 req/일 한도 주의 — 종목당 1~2콜로 제한)
   - 없으면 WebFetch로 Twelve Data 사용: `https://api.twelvedata.com/time_series?symbol={SYM}&interval=1day&outputsize=60&apikey={키}` — CCI는 응답 OHLC로 직접 계산 (TP=(H+L+C)/3, 20기간 SMA와 mean deviation, CCI=(TP−SMA)/(0.015×MD)). 키는 사용자에게 요청하거나 환경 파일에서 읽는다. 키를 코드/파일로 커밋하지 않는다.
3. 종목별 최신 뉴스 2~3건 수집: Alpha Vantage MCP의 NEWS_SENTIMENT, 또는 WebSearch. 레버리지 ETF는 기초지수 뉴스가 중요 — SOXL→반도체(SOX), TQQQ/QLD→나스닥100, SPXL→S&P500, BITX→비트코인 뉴스를 함께 본다.
4. 룰 기반 판정 (사용자 정의 전략, 아래 룰을 그대로 적용):
   - CCI(20)가 -50 ~ -100 구간 & 미보유 → **"진입 구간 도달"**
   - CCI < -100 → **"과매도 구간"**
   - 보유 중 & 현재가 ≤ 다음 미체결 레벨 가격 → **"다음 분할매수 레벨 도달"** (레벨 가격 = startPrice × (1 − level × stepPct/100))
   - 보유 중 & 현재가 ≥ 평단 × (1 + targetPct/100) → **"목표 매도가 도달"** (평단 = fills의 Σ(qty×price)/Σqty)
   - 그 외 → "관망" / "보유 유지 구간"
5. 출력 형식:
   - 최상단: 종목별 1줄 요약 표 (종목 | 종가 | 등락률 | CCI | 판정)
   - 이어서 종목별 상세: 판정 근거 수치, 뉴스 요약(출처 링크 포함), 기초지수 동향
   - 마지막: 오늘 확인할 것 목록 (판정이 "도달"인 항목만)

## 제약 (반드시 지킬 것)

- 이 브리핑은 **사용자가 정의한 룰의 기계적 판정 + 공개 정보 정리**다. "매수하세요/매도하세요" 같은 추천·권유 문구를 쓰지 않는다.
- 주문 실행을 지시하거나 대행하지 않는다. 주문은 사용자가 증권사 앱에서 직접 한다.
- Claude는 라이선스 있는 투자자문업자가 아님을 브리핑 말미에 한 줄로 고지한다.
- 뉴스는 출처를 명시하고, 전망성 내용은 "~라는 관측이 있다" 수준으로 전달한다.
