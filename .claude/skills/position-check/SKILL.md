# 포지션 상태 점검

`data/positions.json`의 포지션과 현재 시세를 대조해 룰 기반 상태를 판정한다. briefing 스킬의 판정 로직만 분리한 수시 호출용 — 뉴스 수집은 하지 않는다.

## 절차

1. `data/positions.json` 읽기. 없거나 오래됐으면(exportedAt이 3일 이상 과거) 웹앱에서 다시 내보내달라고 안내.
2. 보유 종목의 현재가·CCI(20) 조회 (Alpha Vantage MCP 또는 Twelve Data WebFetch — briefing 스킬과 동일 방식).
3. 종목별 계산·판정:
   - 평단 = Σ(qty×price)/Σqty, 수익률 = (현재가−평단)/평단
   - 다음 매수 레벨 = fills에 없는 가장 낮은 level, 가격 = startPrice × (1 − level × stepPct/100)
   - 목표 매도가 = 평단 × (1 + targetPct/100)
   - 판정: 현재가 ≤ 다음 레벨가 → "매수 레벨 도달" / 현재가 ≥ 목표가 → "목표 도달" / 그 외 → 다음 트리거까지 남은 % 표시
4. 출력: 종목별로 [보유수량, 평단, 현재가, 수익률, 다음 매수가(남은 %), 목표 매도가(남은 %), 판정] 표 + 도달 항목 강조.

## 제약

- 매수/매도 추천 문구 금지 — 룰의 기계적 판정만 보고한다.
- 주문 실행 지시·대행 금지.
- Claude는 투자자문업자가 아님을 말미에 고지한다.
